## ☕ Coffee & Code Planner

A retro-styled task planner with a cozy coffee-themed aesthetic. Organize your coding projects and daily tasks in multiple "books" with priority-based task management.

## Features

- **📔 Multiple Task Books**: Create and manage multiple task lists (books) in the sidebar
- **✅ Task Management**: Add, complete, and delete tasks with ease
- **⚡ Priority Levels**: Assign priority levels (High, Medium, Low) to tasks for better organization
- **🎨 Retro Aesthetic**: Pixelated, nostalgic 90s/2000s design with coffee-themed color palette
- **📊 Task Statistics**: View task completion stats at a glance in the header
- **⌨️ Keyboard-Friendly**: Simple and intuitive interface with monospace retro fonts

## Getting Started

### Prerequisites
- Any modern web browser (Chrome, Firefox, Safari, Edge)

### Usage

1. **Open the Planner**: Open `index.html` in your web browser
2. **Create a New Book**: 
   - Enter a name in the "Add New Book" input field at the bottom of the sidebar
   - Press Enter or click to create
3. **Add Tasks**:
   - Select a book from the sidebar
   - Enter your task in the main content area
   - Select a priority level (High, Medium, Low)
   - Click "Add Task" to create the task
4. **Manage Tasks**:
   - Click the checkbox to mark a task as completed/incomplete
   - Click the "×" button to delete a task
5. **Delete Books**: Click the "×" button next to a book name to remove it

## Design

### Color Palette
- **Coffee Dark**: `#2C1810` - Primary dark background
- **Coffee Medium**: `#4A3428` - Secondary accent
- **Coffee Light**: `#8B6914` - Tertiary accent
- **Cream**: `#F4E4C1` - Light background
- **Yellow**: `#FFD700` - Highlight/Active state
- **Paper**: `#FFF8DC` - Content background

### Typography
- **Press Start 2P**: Headers and buttons (retro pixel style)
- **VT323**: Body text and inputs (monospace retro style)

## Technical Details

- **Language**: HTML5 with embedded CSS and JavaScript
- **Storage**: Browser LocalStorage for persistent data
- **Architecture**: Single-page application (SPA) with no external dependencies
- **Responsive**: Flexible layout that adapts to different screen sizes
